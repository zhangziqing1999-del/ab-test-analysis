# AB实验数据分析 — 产品功能、流程与使用说明

## 一、产品定位

本工具面向**多轮交互类产品**（对话式AI、智能推荐、问答助手等）的策略AB实验，提供从原始日志到分析结论的一站式能力。

解决的核心问题：策略上线后，实验组相比对照组，用户的**深入交互意愿**和**内容消费质量**有没有变化？变化是否显著？效果集中在哪类用户问题上？

---

## 二、核心功能

### 功能1：整体效果分析
计算实验组 vs 对照组在核心指标上的差异并做显著性检验：

- **多轮交互转化率**：用户从当前轮追问到下一轮的比例，衡量策略激发"深聊"的能力
- **完读率**：内容被读完的比例，衡量内容质量是否受损

### 功能2：query 智能打标
调用大模型（deepseek-v4-flash + glm-5.1）给每条用户提问自动打两类标签：

- **类型标签**：主观 / 客观
  - 主观：需要判断、评价、推荐，如"推荐几款适合我的车"、"锐评一下雷克萨斯NX"
  - 客观：有确定事实答案，如"速腾裸车价多少"、"极氪7X续航多少"
- **场景标签**：18类汽车子分类（车系推荐、价格、口碑、续航/充电、配置参数、用养成本等）

### 功能3：分维度下钻
在打标基础上，分别看不同类型/场景的query下实验效果如何，定位策略对哪类问题最有效。

---

## 三、分析流程（五阶段）

```
阶段1 预处理+入口分组  →  阶段2 基础AB分析  →  阶段3 LLM打标  →  阶段4 标签维度下钻  →  阶段5 汇总输出
```

| 阶段 | 做什么 | 产出 |
|---|---|---|
| 1 预处理+入口分组 | 提取实验组、按入口分在线/离线、保留全量数据、标记待打标行(time_diff!=0) | 中间文件 `入口分组_<文件名>.xlsx` |
| 2 基础AB分析 | 基于全量数据做无标签整体指标对比 | sheet：AB分析_在线/离线 |
| 3 LLM打标 | 仅对待打标query，去重后并发打标 | 标签写回中间文件 |
| 4 标签维度下钻 | 按query类型、汽车分类拆组分析 | sheet：按query类型/按汽车分类 |
| 5 汇总输出 | 汇总所有结果、清理临时文件 | `AB分析结果_标签维度_<文件名>.xlsx` |

## 四、使用方法

### 输入数据要求

CSV（或已分组的Excel），必需列：

| 列名 | 说明 |
|---|---|
| `ab实验sid` | 复合实验标识，格式 `225457_2-230583_3-…`（`实验ID_组号`，多实验用`-`分隔） |
| `request_type` | 入口类型（区分在线/离线） |
| `rank` | 交互轮次（1=首轮，2=二轮…） |
| `recommend_prompt` | 完读标识，非空且非`""`即完读 |
| `session_id` | 会话ID |
| `time_diff` | 停留/阅读时长（用于筛选有效打标对象） |
| `query` | 用户提问文本 |
| `format_date` | 日期 |

### 运行命令

```bash
python3 scripts/ab_label_analysis.py \
  --file "记忆-数据全部0617-0623.csv" \
  --groups "1,2" \
  --labels "对照组,实验组" \
  --exp-id 225457
```

### 参数说明

| 参数 | 必填 | 说明 |
|---|---|---|
| `--file` | 是 | 原始CSV，或已生成的 `入口分组_*.xlsx`（断点续跑用） |
| `--groups` | 是 | 组号，**第一个为对照组**，逗号分隔，支持2-5组 |
| `--labels` | 是 | 各组中文名，数量与groups一致 |
| `--exp-id` | CSV时必填 | 实验ID，用于从复合 `ab实验sid` 中提取本实验的组号 |

## 五、指标口径

按入口类型锚定不同起始轮次：

### 在线入口（series / pin_dao / dongtai），锚定 rank=1
- **一轮→二轮转化率** = rank2数 / rank1数
- **一轮完读率** = rank1完读数 / rank1数
- **二轮完读率** = rank2完读数 / rank2数

### 离线入口（其余所有 request_type），锚定 rank=2
- **二轮→三轮转化率** = rank3数 / rank2数
- **二轮完读率** = rank2完读数 / rank2数
- **三轮完读率** = rank3完读数 / rank3数

> **为何离线从二轮起算？** 离线入口的首轮(rank=1)绝大多数 `time_diff=0`（曝光但未真正交互即跳出），没有有效打标，从二轮看才有业务意义。

**完读定义**：`recommend_prompt` 非空且非 `""`。

**显著性标记**：

| 标记 | 含义 |
|---|---|
| `***` | 极显著 (p<0.001) |
| `**` | 高度显著 (p<0.01) |
| `*` | 显著 (p<0.05) |
| `ns` | 不显著 |

检验方法：双侧比例Z检验。

## 六、输出与解读

### 输出Excel结构

- `AB分析_在线` / `AB分析_离线`：无标签整体AB
- `按query类型_在线` / `按query类型_离线`：主观/客观下钻
- `按汽车分类_在线` / `按汽车分类_离线`：18个汽车子类下钻
- `标注明细_在线` / `标注明细_离线`：仅含已打标行，全量展示

### 解读步骤

1. **先看整体AB分析** — 判断策略总体有没有效果
2. **再看query类型下钻** — 效果集中在主观还是客观问题
3. **最后看汽车分类** — 细分场景表现（样本<50的子类不建议下定论）
4. **只认带 `*` 的差异** — `ns` 表示两组无实质差别

### 解读注意事项

- **细分维度样本稀释**：下钻到18个汽车子类后，多数格子样本很小，统计功效不足，只能作分布参考。需交叉发现时可聚合成宏类（决策推荐/评价口碑/成本价格/配置参数）。
- **"其他"类占比偏高是正常的**：大量追问query（"好的帮我分析"、"好的继续吧"）无独立车系信息，会被归为汽车分类"其他"，并非打标错误。

## 七、性能与容错

- **去重打标**：先对unique query打标再map回，实测约省52%（11950→5676条）
- **并发+多模型**：CONCURRENCY=8，BATCH_SIZE=50，deepseek/glm轮询
- **断点续传**：checkpoint按unique query维度存储，**仅在全流程结束后清理**；打标中每500条落盘Excel快照。中途被kill重跑可自动从checkpoint恢复
- 整轮约5-6倍提速

## 九、避坑提示

实战中踩过并已修复的关键点，迁移到新场景时重点检查：

1. **先确认实验有哪些组**：如实验225457只有组1(对照)和组2(实验)，别想当然填2、3
2. **复合absid必须用 `--exp-id` 提取**，否则筛选为0
3. **组号按字符串比较**：脚本已统一 `.astype(str)`，避免 str/int 不匹配筛空
4. **AB分析分母用全量数据**：打标只对time_diff!=0，但指标分母要含全量rank=1，否则转化率算出>1
5. **离线算的是二轮→三轮转化**(rank3/rank2)，不是从一轮起算
6. **超大CSV先预处理再打标**，避免直接全量喂入导致超时/OOM
